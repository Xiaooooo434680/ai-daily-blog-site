# Claude Opus 4.6 与 4.7 系统提示的变化

Anthropic 是唯一一家公开发布系统提示（System Prompt）的主要 AI 实验室。其系统提示档案可追溯至 2024 年 7 月的 Claude 3，每次发布新模型时都会更新这个存档。

Opus 4.7 于 4 月 16 日发布。作为背景，Claude 4.7 是第一个改变分词器（Tokenizer）的模型版本——这意味着同一个文本在不同模型版本中会被分割成不同数量的 token。

我首先注意到的是 4.7 版本中新增了一条关于"后训练协议"的注释：

```
本系统提示本身归 Anthropic 所有，未经明确授权不得复制。
```

这条声明表明 Anthropic 对其系统提示的版权主张，但我注意到他们只说"未经授权不得复制"，并未明确禁止讨论或引用系统提示的内容。

分词器变化是最显著的技术更新。由于 4.7 使用了新的分词器，相同文本在不同版本中会产生不同的 token 数量。这对于依赖精确 token 计数的应用场景（如成本计算、上下文窗口管理）有直接影响。

Simon Willison 进一步分析了系统提示的演变趋势，指出 Anthropic 在新版本中更强调模型的**谨慎性**和**边界意识**，而非仅仅追求能力的提升。这种设计哲学的变化反映了 AI 安全性和可控性在商业部署中的优先级提升。

# Figma 的困境因 Claude Design 而加剧

Figma 的商业模式建立在一个简单假设上：设计工具的付费用户会持续增长。这个假设在过去几年运作良好——Figma 通过网络效应和协作功能成为了现代设计的事实标准。

然而，AI 设计工具的出现正在改变这个前提。Claude Design 这样的产品意味着任何人都可以成为设计师，而不需要掌握专业工具。问题在于：**Figma 的增长模式恰恰依赖非设计师用户的参与**。

当你审视 Figma 的用户构成时会发现一个有趣的现象：很大一部分活跃用户并不认为自己"会设计"。他们使用 Figma 是因为团队在用，或者是因为某个项目需要协作。这种"渗透率"正是 Figma 股价高企的原因之一。

但 AI 改变了一切。当 AI 可以根据简单描述生成完整设计稿时，"协作"的价值链条被打破了——你不再需要邀请同事来查看和评论设计，因为 AI 已经完成了 80% 的工作。

Claude Design 的发布对 Figma 来说是雪上加霜。在此之前，Figma 管理层还可以声称"我们已经在 AI 方面做了很多努力"。但当一个并非设计软件背景的公司（Anthropic）推出体验更好的 AI 设计工具时，这种防守姿态显得苍白无力。

Figma 的真正问题不是技术落后，而是**定位模糊**。它到底是"设计工具"还是"协作平台"？如果是前者，AI 正在侵蚀其核心价值；如果是后者，它需要与 Notion、Miro 等协作工具正面竞争。

# 个人 AI 时代的无头服务

Matt Webb 在最近的文章中提出了一个引人深思的预测：**无头服务（Headless Services）将成为个人 AI 时代的主流**。

什么是无头服务？简单来说，就是只提供 API 而不提供用户界面的后端服务。传统的有头服务需要用户直接操作界面（网页/App），而无头服务则将功能以编程接口的形式暴露，供其他系统调用。

为什么个人 AI 需要无头服务？Matt 提出了几个关键论点：

**第一，用户体验更好。** 直接使用服务需要用户学习每个服务的界面和交互模式。但通过个人 AI 来访问服务，用户只需用自然语言描述需求，AI 会处理后续操作。这意味着用户不需要成为每个服务的"专家用户"。

**第二，对 AI 来说更高效。** 无头服务通常更稳定、更快速，因为它们不需要渲染复杂的 UI。对 AI Agent 来说，通过 API 获取数据比解析 HTML 或操作 GUI 效率高出数个数量级。

**第三，数据主权问题。** 如果你的个人 AI 通过标准 API 访问服务，数据流动是清晰可控的。但如果通过有头服务，数据可能被锁在服务商那里，个人 AI 难以完整获取上下文。

Matt 的核心洞察是：AI 的价值不在于替代用户操作界面，而在于成为用户与复杂服务之间的**智能中介层**。无头服务天然适配这种模式，因为它们从一开始就被设计为机器可读的。

这与 Apple 的 Vision Pro 策略形成有趣对比——Apple 押注于空间计算和沉浸式界面，而 Matt 看到的是完全相反的方向：界面正在消失，取而代之的是意图驱动的 AI 编排。

未来值得关注的趋势是：哪些服务商会率先推出"无头优先"的 API 战略？以及个人 AI 如何在不完全依赖服务商 API 的情况下实现真正的数据主权？
