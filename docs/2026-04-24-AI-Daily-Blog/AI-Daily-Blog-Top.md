## 1. An update on recent Claude Code quality reports

**推荐理由**：Anthropic 罕见公开承认 Claude Code 质量事故，披露三个技术根本原因，对 AI 开发者有重要警示意义。

**来源**: Anthropic Engineering

**发布时间**: 2026-04-23

---

### 完整正文

# Anthropic 关于 Claude Code 近期质量报告的更新

过去两个月，我们收到了大量用户反馈，反映 Claude Code 提供的输出质量有所下降。经过深入调查，我们确认这些投诉是有根据的——用户遇到的问题确实存在。

经过全面排查，我们发现问题的根源不在于底层模型本身。Claude 模型家族持续保持高质量标准。但我们发现了 Claude Code 工具链中存在的**三个独立技术问题**，这些问题在特定复杂工作流场景下会直接导致输出质量下降。

第一个问题涉及工具调用状态管理。当用户执行涉及多个连续工具调用的复杂任务时，状态同步机制出现了边缘情况，导致某些工具调用返回的结果未被正确传递到后续处理流程。

第二个问题与上下文窗口管理有关。在处理长对话历史时，重要的上下文信息有时会被不正确地压缩或丢弃，特别是在对话中包含大量文件操作和代码修改的场景。

第三个问题出在错误恢复逻辑上。当工具调用遇到错误时，系统有时会使用不完整的上下文信息进行重试，而不是等待获取完整的错误状态。

这三个问题在单独发生时影响较小，但在复杂的多步骤任务中会累积放大，显著影响用户体验。

我们已经识别并修复了这些问题。所有修复都已通过验证测试，覆盖了之前遗漏的边缘情况。作为预防措施，我们还增强了持续集成测试套件，以更好地捕捉类似问题。

我们向所有受影响用户诚挚道歉，并承诺将继续投资于 Claude Code 的质量和可靠性。

## 2. A pelican for GPT-5.5 via the semi-official Codex backdoor API

**推荐理由**：GPT-5.5 完整评测，Simon Willison 第一时间深度体验，揭示 API 发布延迟背后的技术细节。

**来源**: Simon Willison

**发布时间**: 2026-04-23

---

### 完整正文

# 通过半官方 Codex 后门 API 使用 GPT-5.5

GPT-5.5 正式发布了。它已在 OpenAI Codex 上线，并正在逐步向付费 ChatGPT 订阅用户推送。我获得了预览权限并进行了深度测试。

简单来说：这是一款快速、高效、高度 capable 的模型。正如近期这类模型的通常表现，很难用语言描述它具体好在哪里——我让它构建什么，它就精准地构建出什么。

今天发布中有一个值得注意的遗漏：API 接口。

官方 API 尚未发布。这很不寻常。Sam Altman 在发布视频中暗示 API 会在"不久后"推出，但具体时间线尚不明确。

不过，我发现可以通过 Codex 的半官方 API 端点访问 GPT-5.5。这个"后门"并非全新——它一直是 Codex 的核心部分——但它确实提供了一种绕过官方 API 发布延迟的方式。

使用方式相当直接。我在自己的博客文章中详细记录了具体步骤。简而言之，你需要向特定的 Codex 端点发送请求，使用与 ChatGPT 界面相同的认证机制。

我测试了几个关键场景：

首先是代码补全和重构任务。GPT-5.5 表现优异，能够准确理解复杂需求并生成高质量代码。

其次是多步骤推理任务。在需要多轮思考的问题上，模型展现出色的逻辑连贯性。

第三是对话式编程助手场景。这是 Claude Code 的核心用例，GPT-5.5 在理解上下文和保持任务连续性方面表现突出。

总体而言，这是一款值得期待的模型。唯一的遗憾是官方 API 的延迟发布——这对需要程序化接入的开发者来说确实不便。

## 3. Unauthorized Users in Discord Group Had Weekslong Access to Anthropic's Supposedly-Super-Dangerous Claude Mythos Model

**推荐理由**：Anthropic 机密模型泄露事件，暴露 AI 安全边界的严峻现实。

**来源**: Bloomberg

**发布时间**: 2026-04-23

---

### 完整正文

# Discord 私密群组未经授权用户数周内可访问 Anthropic 据称"极度危险"的 Claude Mythos 模型

据知情人士和 Bloomberg News 查看的文件显示，一群未经授权的用户已获取 Anthropic PBC 新型 AI 模型 Mythos 的访问权限——而 Anthropic 方面曾表示该模型能力强大到"可实现危险网络攻击"。

这些用户在一个私人在线论坛上获得了 Mythos 的访问权限。值得注意的是，他们获得访问的日期与 Anthropic 首次宣布向有限数量的公司开放模型测试的计划是同一天。这意味着未经授权的访问持续了数周时间。

Anthropic 在其公开声明中将 Mythos 描述为一项技术突破，但同时强调其具有潜在的危险性。公司表示，Mythos 的能力已达到可辅助实施复杂网络攻击的水平，这也是为什么公司选择只向经过严格筛选的合作伙伴提供访问。

然而，此次泄露事件暴露了 Anthropic 在模型发布控制流程中的漏洞。Bloomberg 报道称，Anthropic 原本计划的"有限发布"实际上未能有效阻止未经授权的访问。

安全专家对此事件表示深切关注。一家大型网络安全公司的研究员表示："如果 Anthropic 对 Mythos 能力等级的评估是准确的，那么未经授权的访问可能会带来严重后果。模型一旦流出，很难追踪其最终用途。"

Anthropic 拒绝对具体的安全措施发表评论，但在一份声明中表示公司"严肃对待模型安全"，并正在"持续评估和改进我们的发布协议"。

此事件发生在 AI 行业对前沿模型安全管控日益关注的大背景下。多个 AI 实验室已承诺采取更严格的发布前评估和访问控制措施，但此次 Mythos 泄露事件表明，承诺与执行之间可能存在显著差距。

目前尚不清楚这些未经授权的用户具体用 Mythos 做了什么，也不清楚泄露范围有多大。但可以确定的是，Anthropic 面临着解释为何其"严格控制"的发布机制未能阻止未经授权访问的压力。
