# AI 的经济学说不通

**作者：Ed Zitron**

生成式 AI 行业正在面对一个它不愿承认的事实：经济上它根本不成立。

让我们从最基本的开始。OpenAI 的 ChatGPT Plus——每月 20 美元的订阅服务——据 The Information 报道，将从 2025 年的 4400 万用户下降到 2026 年预计的 900 万。那是一个 80% 的跌幅。而 OpenAI 通过推出更廉价的订阅层级来填充这个数字，但这只是用低价值客户替换高价值客户。

与此同时，AI 服务的实际运营成本持续飙升。GitHub Copilot 的周环比成本自 1 月以来几乎翻了一番。Uber 在 2026 年仅仅几个月内就用完了全年的 AI 预算——其中 Claude Code 的使用量激增是主要原因。

这些数字背后的数学很简单：每个订阅用户消耗的计算成本远超他们的订阅费。Anthropic 每收 1 美元订阅费就要烧掉 8 到 13.5 美元。微软在 GitHub Copilot 上的支出可能是其收入的十倍。

这不是一个可以通过"扩大规模"解决的问题。更大的规模意味着更大的损失。AI 模型每次推理都需要大量 GPU 算力，而 GPU 并不便宜——即使 NVIDIA 正在销售比全球数据中心实际能够安装的数量更多的芯片。

当风险投资的补贴最终退潮时——它终将退潮——整个行业将面临一场灭绝级事件。到 2026 年底，我预计大多数 AI 服务将转向基于代币的计费模式，这将进一步压缩用户群。

AI 行业建立在无限便宜算力的假设之上。但现在我们正撞上物理和经济的双重现实之墙。

# Ghostty 离开 GitHub

**作者：Mitchell Hashimoto**

我决定将 Ghostty 终端模拟器项目搬离 GitHub。

这不是一个轻率的决定。作为 HashiCorp 的联合创始人，我深知平台选择对开源项目的影响。但经过深思熟虑，我认为对于 Ghostty 的未来，这一步是必要的。

原因有几个。首先，对 GitHub 的过度依赖让整个开源生态系统变得脆弱。当 GitHub 宕机时——它确实会宕机——全球的开发工作流陷入停滞。当一个平台承载了如此多的开源基础设施，我们实际上把所有鸡蛋放在了一个篮子里。

其次，GitHub Actions 的可靠性问题日益令人担忧。作为 CI/CD 系统，它在关键时刻的不稳定性已经影响了开发效率。虽然 GitHub 在努力改进，但根本问题在于：一个托管服务不可能对所有项目都最优。

第三，也是最重要的，我认为开源社区需要多元化的托管生态。GitHub 的成功部分建立在网络效应上——所有项目都在那里，所以更多项目选择去那里。这是一个自我强化的循环，但它也阻碍了创新。

我不确定 Ghostty 的下一站在哪里，但我确信这个决定是正确的。开源的本质是自由——包括选择你的工具和平台的自由。

# (一个)好的 AI 已经到来

**作者：Anil Dash**

关于 AI 的文化战争在过去几年中沿着可预测的分裂线展开。批评者正确地指出了大型 AI 平台的问题——从劳工剥削到版权侵犯再到环境影响。但在这场战役中，我们忽略了另一个故事。

那个故事是：(一个)好的 AI 已经存在。

我特意用了"一个"——单数——因为好的 AI 往往不是那些占据头条的通用大模型。它是专注的、领域特定的、目标有限的。它在无声中工作，解决那些大型科技公司不屑于解决的问题。

在医疗诊断中，AI 模型正在帮助放射科医生发现早期癌症——不是替代医生，而是增强他们的能力。在语言保护中，AI 正在帮助濒危语言社区创建数字资源，让下一代能够学习祖先的语言。在无障碍领域，AI 驱动的工具正在让互联网对残障人士更加友好。

这些不是价值数十亿美元的产品。它们不会制造出令人震撼的 CEO 主题演讲。但它们确实在改善真实人类的生活。

"好的 AI"不是关于 AGI 或超级智能。它是关于工具——安静、专注、有用的工具——在特定环境中被谨慎部署，有明确的用途，并在适当的限度内运行。

AI 的未来不必是巨头的叙事。它可以是——应该是一百万个小应用，每个都让某人的世界变得稍微好一点。
