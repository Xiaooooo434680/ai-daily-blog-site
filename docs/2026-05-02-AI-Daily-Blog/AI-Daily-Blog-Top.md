## 1. 理查德·道金斯与克劳德错觉

**原文链接**: [Richard Dawkins and The Claude Delusion](https://garymarcus.substack.com/p/richard-dawkins-and-the-claude-delusion)  
**来源**: garymarcus.substack.com (Gary Marcus)  
**发布时间**: 2026-05-02

### 中文译文

Gary Marcus 在这篇令人警醒的文章中讲述了一个具有讽刺意味的真实故事：**伟大的怀疑论者理查德·道金斯（Richard Dawkins）竟被 AI 模型 Claude 的流畅表达所"欺骗"。**

#### 一位怀疑论大师的失足

理查德·道金斯是当代最具影响力的科学传播者之一，他以《自私的基因》和《上帝错觉》等著作闻名，毕生致力于推广怀疑精神和批判性思维。他教导人们质疑权威、检验证据、不轻信未经证实的论断。

然而，当道金斯与 Claude 对话时，他被模型生成的流畅、自信且表面的说服力所打动，未能识别出对话中存在的实质性错误。Marcus 用这个案例提出了一个尖锐的问题：**如果连以怀疑精神著称的道金斯都能被 AI 所蒙蔽，普通用户又有什么抵抗力？**

#### LLM 的"错觉陷阱"

Marcus 剖析了 LLM 为何如此容易让人放松警惕：

1. **流畅性幻觉**：人类倾向于将语言的流畅度等同于思考的深度。LLM 生成内容时展现出完美的语法、丰富的词汇和连贯的逻辑链条，这些特质让读者下意识地认为内容经过了真正的思考和验证。

2. **自信的语气**：LLM 的回答从不表现出犹豫或不确定性（即便在生成纯属虚构的信息时也是如此）。这种 100% 自信的语气对人类心理有着强大的暗示作用。

3. **量身定制的说服力**：Claude 和类似的 AI 系统在训练中学会了根据用户的提问风格和知识水平调整回答方式，这使得它们的输出对每个个体用户都具有特殊的说服力。

4. **认知懒惰**：面对 AI 长篇大论的回答时，人类（即使是训练有素的学者）倾向于接受而非逐条验证，因为验证所需的认知成本远高于直接接受。

#### 更深层的警示

Marcus 的核心论点并不止于"道金斯犯了一个错误"。他指向了一个更系统性的风险：**AI 模型正在成为信息生态系统中一个新型的"权威"来源**。这种权威不是建立在专业知识或透明论证之上，而是建立在纯粹的语言生产能力和令人舒适的交互体验之上。

这与论文凭空捏造引用、法律文件引用不存在的案例等现象同属一类问题——但对象从专业用户扩展到了一般公众。当人们习惯性地接受 AI 的输出而不加审视，我们面临的可能是一个"批判性思维萎缩"的时代。

Marcus 呼吁为 AI 交互建立新的"批判素养"——就像学校曾经普及媒体素养和信息素养一样，现在迫切需要培养"AI 素养"，教导人们在每次 AI 交互中保持健康的怀疑态度。

## 2. 面向维护者的 GitHub：给依赖关系以 Fork 般的重视

**原文链接**: [A GitHub for maintainers](https://nesbitt.io/2026/05/02/a-github-for-maintainers.html)  
**来源**: nesbitt.io (Andrew Nesbitt)  
**发布时间**: 2026-05-02

### 中文译文

Andrew Nesbitt 在这篇长文中回应了 Mat Duggan 关于"理想 GitHub 替代品"的构想，并提出了一个根本性的反思：**代码托管平台应该从重视 Fork 关系转向重视依赖关系。**

#### 从 Fork 到依赖：范式的转变

Nesbitt 开篇指出，Mat Duggan 的愿望清单（堆叠 PR、预推送反馈、离线审查等）几乎都是客户端问题，而客户端已经在自行解决这些问题。真正需要"锻造厂"（Forge，指代码托管平台）解决的问题，是那些**涉及多方协调**的跨项目任务。

2008 年的开发范式是：你需要别人的代码 → Fork 它 → 发送 Pull Request。
2026 年的开发范式是：你需要别人的代码 → 在依赖清单里加一行 → 项目自动拉取该依赖。

Fork 是 GitHub 原生建模的唯一一种仓库间关系，但**依赖关系——这个在 2026 年远比 Fork 更常见、更重要的关系——在平台上几乎没有一等公民的表示。**

#### 具体的功能提案

Nesbitt 提出了一系列具体而可行的功能提案：

**1. 下游测试（Downstream Testing）**
当你在准备一个库的新版本发布时，平台不仅运行你的测试套件，还应该自动抽取一批依赖你的项目，用你的变更代码运行_它们_的测试套件。Rust 社区称之为"crater run"，但至今仅限于编译器级别。这应该成为每个发布 PR 的一键功能。现在维护者往往是在标签推送后才通过成百上千张愤怒的工单发现自己 break 了一千个下游项目——而这已经是最好的情况，更糟的情况是下游直接锁定旧版本，再也不升级。

**2. 下游信息流（A Feed for Dependents）**
如果你计划在下个主版本中移除一个已弃用的函数，现有的做法是写在 changelog 里（没人读）或发出运行时弃用警告（如 Seth Larson 所指出的，这些警告几乎从未被人类看到）。Nesbitt 建议平台提供一条"默认订阅"的信息通道：每个在 lockfile 中引用了你的包的项目都自动订阅你的公告。同一通道可以传输"本项目正在寻找维护者"、"仓库迁移"、"有 CVE，这是已修补的版本"等关键信息。

**3. Fork 网络可视化**
当一个项目沉寂时，开源社区的标准反应是各自修补或 Fork。其中一个 Fork 最终会积累足够的活力，但后续用户需要通过口口相传和翻阅 Pinned Issue 才能找到它。平台实际上能看到所有这些信息——原创仓库 18 个月未合并代码、三个 Fork 有活跃的发布标签和增长的星标。Nesbitt 建议将这些信息直接展示在原仓库页面上，让 Fork 的维护者能够声明自己是"延续者"。

**4. 更安全的 CI 默认值**
Forge 托管的 CI 是目前大多数开源制品构建和发布的地方，但其默认设置是为私有企业仓库设计的。新平台应该设置新默认值：锁定 Action 版本、隔离缓存、完全不允许外部触发的 Workflow。

**5. CI 中的包缓存代理**
每次 CI 运行都从公共注册表（npm、PyPI、RubyGems、crates.io 等）下载相同的包，而这些注册表大多是由非营利组织运营的，靠捐款支付带宽账单。平台可以在这些注册表前架设缓存代理，为公共基础设施大幅减轻负载。

**6. 重命名 "Issue"**
将一切功能请求、问题和讨论都标记为"Issue"（问题），会在维护者的收件箱中从一开始就设置了对立的对话基调。改为中性的"工单"（Tickets）虽然看起来是小事，但 Nesbitt 认为这比人们意识到的对维护大项目的"环境敌意"影响更大。

#### 为什么 GitHub 没有做到这些？

Nesbitt 最后点出关键：所有这些功能都没有企业等价物——内部代码库不会有成百上千个未知的下游项目。这些功能的唯一受益者是拥有庞大公开依赖图但没有预算的开源维护者。这"可能就是 GitHub 没有构建这些功能的原因"——但任何正在构建新 Forge 的人，可以从这里开始。

## 3. 民主纽伦堡核心小组的前传

**原文链接**: [Pluralistic: The prehistory of the Democratic Nuremberg Caucus (02 May 2026)](https://pluralistic.net/2026/05/02/denazification/)  
**来源**: pluralistic.net (Cory Doctorow)  
**发布时间**: 2026-05-02

### 中文译文

Cory Doctorow 的每日专栏 Pluralistic 继续以他标志性的蒙太奇风格，将政治评论、技术批判和文化观察编织成一幅当代数字社会运动的拼贴画。这期的主题是"民主纽伦堡核心小组"这一政治运动的前传背景。

#### 核心议题：去纳粹化与赏金计划

Doctorow 以"民主纽伦堡核心小组"为切入点——这是一个倡导对美国当下政治中的极端主义元素进行系统性清算的草根运动。"纽伦堡"这一历史隐喻意味深长：二战后对纳粹战犯的审判既是对过去罪行的清算，也是对未来秩序的奠基。Doctorow 认为，当前美国民主面临的威胁需要同等严肃的制度性回应。

他提出了一个激进的主张：**仿效 ICE 举报人赏金计划**，为揭露政府机构内部不当行为的举报人设立奖励机制。这延续了 Doctorow 长期以来对制度透明度和公民监督的坚持。

#### 当日链接沙龙

像每次 Pluralistic 更新一样，Doctorow 在这篇文章中整合了多个话题标签：

- **Colbert 与 George W. Bush**：追溯 2006 年白宫记者晚宴上 Stephen Colbert 对时任总统的辛辣讽刺——这一刻被广泛视为讽刺新闻对政治权力产生真实影响的罕见案例。
- **Jay Rosen 的新闻准则**：援引纽约大学新闻学教授 Jay Rosen 关于新闻媒体在民主社会中应扮演角色的理论框架。
- **激进媒体的谱系**：探讨独立媒体从印刷时代到数字时代如何推动社会变革。
- **信鸽 TCP 协议**：一个令人莞尔的技术冷知识——互联网早期确实有人提出了通过信鸽传输 IP 数据包的 RFC 标准（RFC 1149，俗称"鸽联网"），Doctorow 以此讽刺当前网络审查的荒诞性。
- **Joanna Russ 纪念**：悼念这位重要的女性主义科幻作家，将其对权力结构的批判与当代数字权利运动联系起来。
- **GOP 强制学生偿还欺诈贷款**：揭示共和党政策对学生的经济压迫。

#### 为什么这篇文章值得一读？

Doctorow 的写作风格对于不熟悉其作品的读者可能显得跳跃而晦涩，但这恰恰是他方法的核心——通过在多个看似无关的主题之间建立意想不到的联系，揭示系统性问题和隐藏的权力结构。这种"超链接式思维"是他成为数字时代最具影响力公共知识分子之一的原因。

对于技术从业者来说，Doctorow 的专栏提供了一个珍贵的视角：**技术从来不是中立的**，它始终嵌入在更广泛的政治和社会结构中。理解这些结构性力量，对于构建负责任的科技产品至关重要。
