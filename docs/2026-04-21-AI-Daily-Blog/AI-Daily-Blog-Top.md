# Anthropic 短暂移除 Claude Code 的 Pro 订阅权限

![Header](https://storage.ghost.io/c/24/d8/24d8fcec-dfba-4f40-a467-823d2941ae46/content/images/size/w600/2024/01/wide-with-letters-1.jpeg)

**作者：Ed Zitron**

2026 年 4 月 21 日下午，Anthropic 从各个定价页面中移除了 Claude Code 对其 $20/月 "Pro" 计划的访问权限。现有 Pro 用户似乎仍可通过 Claude Web 应用使用该工具，但 Claude Code 的支持文档在短时间内仅提及了 "您的 Max 计划"。

Anthropic 的 Amol Avasare 在 Twitter 上称这是一次 "仅针对 2% 新个人用户的小测试"。但这并不太说得通，因为所有支持文档和 Claude 网站都显示 Pro 用户没有访问权限。

此前，Pro 级订阅用户可以通过命令行界面、Web 应用和桌面应用使用 Claude Code。现在他们很可能需要为 API 访问付费才能继续使用该工具。

一份 4 月 10 日的存档支持页面标题为 "在您的 Pro 或 Max 计划中使用 Claude Code"，而现在的版本已改为 "在您的 Max 计划中使用 Claude Code"。

![定价页面截图（桌面版）](https://storage.ghost.io/c/24/d8/24d8fcec-dfba-4f40-a467-823d2941ae46/content/images/2026/04/Screenshot-2026-04-21-at-3.35.28---PM.png)

![定价页面截图（移动版）](https://storage.ghost.io/c/24/d8/24d8fcec-dfba-4f40-a467-823d2941ae46/content/images/2026/04/image-1.png)

Anthropic 的定价页面（桌面端和移动端）均显示 Claude Code 不再出现在 Pro 层级中。

这位记者也询问了 Avasare，如果这只是一个测试，为什么支持文档和网站会反映 Pro 用户没有访问权限。他没有回应。他确实说了公司 "在此过程中做了一些小的调整"，并且 "我们当前的计划并非为此构建的"——暗示将出现更大的变化。

目前尚不清楚这一变化是追溯应用于现有 Pro 用户、仅影响新注册用户，还是 Anthropic 计划最终从所有 Pro 账户中移除 Claude Code。最终，这些变化在某个未知时间点被逆转。我将保留这篇文章作为所发生事情的记录。

# AI 末日四骑士

![Header](https://storage.ghost.io/c/24/d8/24d8fcec-dfba-4f40-a467-823d2941ae46/content/images/size/w600/2024/01/wide-with-letters-1.jpeg)

**作者：Ed Zitron | 阅读时间：22 分钟**

我已经花了大约四年时间撰写关于 AI 泡沫的宏大叙事。但这篇文章不同——我将通过这些正在发生的奇怪事件，引出我的"灰马"概念，作为泡沫正在破裂的警示信号。

### 第一骑士：Anthropic 产品因容量限制频繁崩溃

Anthropic 应该停止接收新客户，直到容量问题得到解决。在科技行业，"四个九"（99.99%）的正常运行时间是标准实践。然而 Anthropic 的 Claude 聊天机器人在过去 90 天内的正常运行率仅为 98.79%，API 为 99.09%，Claude Code 为 99.25%。

98.79% 的正常运行率意味着每年大约有四天半的停机时间。Claude for Government 的正常运行率为 99.91%，而政府服务通常要求至少四到五个九。

据《华尔街日报》报道，"自 2 月中旬以来，Anthropic 各系统的中断变得如此普遍"，一些企业客户正在更换服务商。Retool 创始人 David Hsu 表示，他最近转而使用 OpenAI 的模型，因为 "Anthropic 一直在宕机"。

Opus 4.6 也存在严重的性能问题。AMD 高级 AI 总监 Stella Laurenzo 分析了 6,852 个 Claude Code 会话文件，记录了从 "研究优先行为到编辑优先行为的可测量转变"。关于 Opus 4.7，Reddit 上一篇获得 2,300 赞的帖子标题写道 "Claude Opus 4.7 是一次严重的退步，而非升级"。

### 第二骑士：Anthropic 的容量问题无解

Anthropic "无法为其客户提供稳定可靠的服务"。公司的扩张计划涉及与 Broadcom 的交易 "从 2027 年开始上线"，与 Hut8 的理论容量合作，以及 CoreWeave——后者截至 2025 年底只有约 850MW 的活跃电力容量。

世界可能正在 "触及可用 AI 算力的绝对物理极限"。即便增加容量，问题依旧：是为了改善现有客户的服务？还是为更多客户腾出空间？还是支持更大的模型？增加容量仅仅意味着 "产品以更高的成本稍微好一点地工作"。

### 第三骑士：Anthropic 的增长故事是"骗局"

Anthropic 的增长建立在每 1 美元订阅收入就要烧掉 8 到 13.5 美元的订阅模式之上。风险投资的补贴、媒体报道和缺乏监管使这一切成为可能。如果 Anthropic "被迫按实际成本收费"，其增长将崩溃。结论是："Anthropic 是一个骗局。"

### 第四骑士：代币计费时代来临

微软计划暂停 GitHub Copilot 新个人用户注册、收紧速率限制、从 $10/月层级中移除 Opus 模型，并转向代币计费。GitHub Copilot 的周环比成本 "自 1 月以来几乎翻了一番"。微软停止接收新 Copilot 付费订阅——这是 "软件历史上最令人震惊的举动之一"。

到 2025 年底，GitHub Copilot 收入约为 10.8 亿美元，而微软可能花费 "高达十倍" 于此的金额。Anthropic 已在两周前将企业客户转为按代币计费。预测：到 2026 年底，"大多数 AI 服务将把部分或全部客户转为代币计费"。

### AI 成本逼近人力成本的 10%

高盛报告显示，公司 "在推理方面的初始预算超出数量级"，推理成本 "逼近人力成本的约 10%"，并可能在接下来几个季度达到与人力成本持平。

Uber 因 Claude Code 的使用量暴增，"在 2026 年仅仅几个月内就用完了全年 AI 预算"。Uber CTO 指出 "11% 的真实后端系统代码更新是由 AI 代理编写的"。

### 结语

Zitron 批评高管们以脱离现实的方式讨论 "AI 代理"。他指出："生成式 AI 深度不盈利，似乎没有提供切实的生产力提升"，对企业和客户都在亏钱。

整个 AI 行业包括 OpenAI 和 Anthropic 的理论收入 "去年约为 650 亿美元"。创始人 Scott Stevenson 声称 "超过 50% 的企业 AI 初创公司" 使用 "合同化 ARR" 通过含退出条款的多年合同来虚增估值。

AI 行业未来的 "完全建立在无限资源的假设之上"，大多数 AI 公司只是 "Anthropic 和 OpenAI 模型的前端界面"。顶层是 NVIDIA，"销售的 GPU 数量超过了实际能够安装的数量"。他警告一场 "投资界和主流媒体的合法性危机" 即将到来。

# AI 奥德赛第四部：令人惊叹的编码代理

**作者：Wayne Joubert（发表于 John D. Cook 的博客）**

AI 编码代理在去年夏天有了显著改进，接着在去年 12 月到今年 1 月又再次大幅提升。以下是我自上一篇相关文章以来的使用体验。

这些模型在主观感受上明显更加智能。它们能够完成范围更广的任务。它们似乎对代码库以及你在做什么有了更大、更全面的深度理解。

### 生产力提升

在 2025 年 8 月，我估计 AI 辅助了约 20% 的编码工作。而截至 2026 年 4 月，这一比例大约在 60%。

### 局限性

编码代理尽管强大，但仍有不少局限：

- **需要引导**：它们有时需要被告知在哪里查找信息
- **只见树木不见森林**：可能高度优化局部细节，却忽略了更大的图景
- **过度优化**：偶尔会生成针对测试适配器过度优化的代码，而非满足实际使用场景
- **风格不一致**：有时生成的代码与现有代码库在概念上不完全一致
- **代码冗余**：倾向于生成比必要更多的代码。不过，我也成功使用代理重构并压缩了它们自己生成的代码，有一次将代码量减少了一半以上

### 工具选择与使用方式

我使用的是 OpenAI Codex 而非 Claude Code，并且一些技术上有公信力的人对此选择表示了认可——这让我感到鼓舞。

我的工作是一个研究项目，其中 "代码本身就是研究产物"。这使得编码成为一个发现过程，而不是基于规格的工程任务。我始终保持人类可读的代码库，并积极与代理协作，而非将其视为 "黑箱软件工厂"。

### 技能退化问题

关于 "不用 AI 就无法编码" 的担忧，我认为长期积累的编程经验使这种情况不太可能发生。同时我也注意到，代理有时会教我一些新的编码惯用语。我写道："我认为我永远不会停止写代码，就像我永远不会停止在信封背面涂鸦随机的想法一样。"

### 多代理尝试

我仍在努力管理同时运行多个代理的认知负荷，但正在尝试在其中一个代理工作时启动另一个，以保持生产效率。

### 10 倍效率的实际案例

最近，我需要为代码添加一项新的求解能力——使用来自不同 Python 包的不同数学求解器。这项任务通常需要我花费大约两个月的时间来学习一种全新的算法方法和相关库。

借助编码代理，我在四天内完成了这项工作——新增了数千行代码。正确性通过已有的测试套件得到验证，这些测试用例涵盖了旧求解器和新增求解器下的广泛场景。

### 成本

我使用 OpenAI Plus 计划（$20/月），始终保持在用量限制之内，并有意识地使用上一代模型来控制成本。当被问及更多细节时，我确认代码量约数千行、正确性通过测试套件验证、代码人类可读，并且我需要提示代理才能生成文档。
