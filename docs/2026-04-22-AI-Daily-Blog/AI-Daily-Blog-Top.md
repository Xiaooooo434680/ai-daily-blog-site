# Is Claude Code going to cost $100/month? Probably not - it's all very confusing

4 月 22 日，Anthropic 悄然（实际上是"沉默地"，没有任何公告）更新了 claude.com/pricing 页面，在 Claude Code 的定价中添加了一个微小但关键的细节。与此同时，Anthropic 从 $20/月的 Pro 计划中移除了 Claude Code 的访问权限——至少在各种定价页面上如此。现有的 Pro 用户似乎仍然可以通过 Claude 网页应用访问。

这个变动立即在开发者社区引发轩然大波。用户开始担心每月 $100 的传言是否成真。

但情况很快发生了逆转。几个小时后，Anthropic 撤回了这一变动，Claude Code 重新回到了 Pro 计划的覆盖范围内。这整个混乱事件被 Simon Willison 全程记录下来。

这起事件的讽刺之处在于：Anthropic 是唯一一家在系统提示变更时发布正式公告的主要 AI 实验室，却在其产品定价上采取了最不透明的方式。没有人知道他们为什么首先进行了这个改动，也没有人知道为什么他们如此迅速地撤回了它。

对于整个 AI 行业来说，这件事揭示了一个更深层的问题：**我们仍在学习如何为 AI 产品定价**。是按用户数？按 Token 数？按功能？还是有其他的计费维度？这些问题目前都没有标准答案。每家公司都在摸着石头过河，而价格变动往往会引发用户的不安——因为用户已经习惯了"一次订阅、永久有效"的模式。

AI 产品的定价最终会走向何方？可能是基于实际使用量的动态定价，可能是分层订阅，可能是企业级定制合同。无论哪种模式，核心原则都是一样的：**清晰度和可预测性**。而 Anthropic 这次的做法恰好违背了这两条原则。

# Qwen3.6-27B: Flagship-Level Coding in a 27B Dense Model

Qwen（通义千问）发布了其最新模型 Qwen3.6-27B，这是一个具有 270 亿参数的密集（Dense）模型。官方宣称这个模型在编码任务上达到了旗舰级性能，超越了此前的开源旗舰 Qwen3.5-397B-A17B——那个模型采用 MoE（混合专家）架构，总计 3970 亿参数，但只有 170 亿参数处于激活状态。

这个发布对 AI 行业有几点重要意义：

**首先是效率的突破。** 270 亿激活参数对比 170 亿激活参数，Qwen3.6-27B 以相近的激活参数规模实现了对更大 MoE 模型的超越。这意味着**密集模型在特定任务上可以比 MoE 模型更高效**——如果你只需要处理一个任务，密集架构的"全参数参与"反而成了优势。

**第二是开源的胜利。** Qwen3.5-397B-A17B 已经是开源模型中编码能力最强的之一，而 Qwen3.6-27B 在参数更少的情况下实现了超越。这对于需要本地部署的企业来说是个好消息——更小的模型意味着更低的部署成本和更快的推理速度。

**第三是对 Scaling Law 的重新思考。** 传统观点认为"越大越好"，但 Qwen3.6-27B 暗示了另一条路径：**针对特定任务的专项优化可能比单纯追求参数规模更有效**。这个模型显然经过了大量的编码任务微调，其训练数据和处理流程都针对编程场景进行了优化。

Simon Willison 在评论中指出，这个模型的发布证明了 AI 编码能力的进步速度远超预期。一年前，人们还在讨论 AI 能否完成简单的代码补全；现在，同一个家族的模型已经在旗舰级编码任务上与最大规模的 MoE 模型竞争。

# Ben Thompson on Tim Cook's Legacy

Ben Thompson 在 Stratechery 发表了一篇深度分析文章，探讨 Tim Cook 留给苹果的遗产。标题是"Tim Cook 的绝佳时机"——这个"时机"既指 Cook 加入苹果的 1998 年，也指他接替 Steve Jobs 的 2011 年，更指他在供应链管理领域做出的每一个决策。

**Cook 的运营天才**

Cook 加入苹果时，公司的运营——集中在自营工厂和仓库——是沉重的负担。那些工厂和仓库是 Steve Jobs 回到苹果后关闭的，但这个决策的执行者是 Cook。Thompson 写道：

> Cook 方法论地关闭了这些设施，将苹果的制造基地转向中国，创建了一个 JIT（Just-In-Time）供应链体系，年复一年地协调难以置信的规模。

这个转变是革命性的。在 Cook 加入苹果之前，苹果是一个"纵向一体化"的公司——从设计到制造到销售，全部自己来做。这种模式在 80 年代和 90 年代初期还很常见，但随着全球化和亚洲制造成本的下降，这种模式的弊端开始显现：成本高、灵活性差、产能受限。

**JIT 供应链的力量**

JIT 供应链的核心思想是"只在需要的时候、只生产需要的数量"。这需要精确的需求预测、高效的供应商协调、以及强大的物流系统。苹果的供应商网络遍布全球，而 Cook 的天才在于他能够协调这个复杂的网络，使其如同一台精密的机器运转。

这与他在 IBM 和康柏的经历分不开。在加入苹果之前，Cook 已经在 IBM 负责供应链管理 12 年，负责康柏的全球采购和履行业务。他在运营领域的经验是无可替代的。

**真正的遗产**

Thompson 认为，Cook 留给苹果的真正遗产不是某一款产品，而是**一套运营体系**。这套体系让苹果能够以难以置信的规模和速度生产产品，同时保持质量和灵活性。当 iPhone 的需求从零增长到数以亿计时，苹果的供应链经受住了考验——这不是偶然，而是 Cook 设计的体系的必然结果。

有趣的是，当 Trump 在社交媒体上称赞 Tim Cook（把他叫成"Tim Apple"）时，他可能没有意识到自己赞扬的是 Cook 的运营能力而非创新能力。但正是这种运营能力，让苹果成为世界上最有价值的公司之一。
